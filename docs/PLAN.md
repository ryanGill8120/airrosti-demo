# Build Plan

Phased plan for the Airrosti demo. Work top-to-bottom; each phase produces a
demonstrable slice. Mark items complete with `[x]` as they ship.

## Phase 1 — OpenFDA integration (no auth)
- [x] Add `AirrostiDemo.Shared/OpenFda/FdaReactionCount.cs` and `FdaCountResponse.cs` DTOs
- [x] Register `IHttpClientFactory` named client `"openFDA"` with base address `https://api.fda.gov/`
- [x] Add `AirrostiDemo.Server/Controllers/DrugSideEffectsController.cs` with `GET /api/drugsideeffects/{drugName}` returning the reaction-count response
- [x] Add `AirrostiDemo/Pages/DrugSearch.razor` — search box + results table/chart
- [x] Pre-wire 6 demo drugs (ibuprofen, naproxen, oxycodone, acetaminophen, tramadol, celecoxib)
- [x] Verify response shape against live FDA data, then write a brief readme blurb

## Phase 2 — Auth scaffolding
- [x] Add NuGet packages to Server: `Microsoft.AspNetCore.Identity.EntityFrameworkCore`, `Microsoft.AspNetCore.Authentication.JwtBearer`, `Microsoft.EntityFrameworkCore.Sqlite`, `Microsoft.EntityFrameworkCore.Tools`
- [x] Add `AppUser : IdentityUser` and `AppDbContext : IdentityDbContext<AppUser>`
- [x] Wire Identity + JWT in `Program.cs`; key in `appsettings.Development.json`
- [x] Create `AuthController` with `register` + `login` endpoints; login returns `{ token, expiresAt }`
- [x] Add `RegisterDto` and `LoginDto` to `AirrostiDemo.Shared/Auth/`
- [x] Initial migration + DB create

## Phase 3 — Saved reports (protected)
- [x] `SavedReport` entity (Id, UserId, DrugName, ReportJson, SavedAt)

## Phase 5 — Production deployment
- [x] `appsettings.Production.json` (server) + `wwwroot/appsettings.Production.json` (client) for prod URLs / CORS / JWT issuer-audience. `Jwt:Key` deliberately absent — comes from Render env var `Jwt__Key` (generated by Blueprint).
- [x] `render.yaml` Blueprint at repo root — server (Docker web service) + client (static site) both auto-deploy from `master`.
- [x] `AirrostiDemo.Server/Dockerfile` — multi-stage .NET 8 build, listens on `:10000`.
- [ ] Push to GitHub (`https://github.com/ryanGill8120/airrosti-demo.git`) and create the Render Blueprint from the repo (one-time, in Render dashboard).
- [ ] Configure custom domain in Cloudflare DNS, verify Let's Encrypt cert. After domain is live, update `Cors__AllowedOrigins` (server env var) and `wwwroot/appsettings.Production.json` (`ApiBaseAddress`) to the custom hostnames.
- [ ] Smoke test full flow on prod URL (search → register → save report → my reports).
- [ ] Update README with live demo link.

### Phase 5 trade-offs to surface in interview
- SQLite on Render's ephemeral disk: `demo.db` resets on each deploy. Documented in CLAUDE.md. Add a Render Disk if persistence matters.
- Static-site build installs the .NET SDK into the build container on every build (~10s overhead). Acceptable; alternative is a custom build image.
